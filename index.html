CSS position Values

Value

Description

Use Cases

static

Default position value. Elements will follow the normal flow of the document. The top, right, bottom, left and z-index properties have no effect.

Regular elements that don't need any special positioning.

relative

Element follows the normal flow of the document but offset according to the top, right, bottom, and left properties. Other elements will position themselves as if this element was not offset.

Elements that "pop" out when you hover over them.

absolute

Element does not follow the flow of the document, and is positioned relative to its closest positioned ancestor if available; otherwise, it is placed relative to the initial containing block. The top, right, bottom, and left properties can be used to offset the element.

Placing elements whereever you want in a page and overlapping elements.

fixed

Element does not follow the flow of the document and is positioned relative to the initial containing block. Stays in the same place in the viewport as you scroll. It is fixed based on the top, right, bottom, and left properties.

Headers or navigation bars that you want to stay visible as you scroll the page.

sticky

Element is relatively positioned and follows the flow of the document until a threshold is hit. This threshold is specified using the top, right, bottom, and left properties. After the threshold is hit, the element is fixed in the viewport until it reaches the boundary of its parent.

Headings in an alphabetical list.
